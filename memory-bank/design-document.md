# Word Formatter 设计规范（Design Specification）

> Version 2.0

---

# 一、设计目标

Word Formatter 是一款面向教师、学生、公务员、编辑等用户的 Windows 桌面软件，用于批量统一 Word 文档格式，并支持模板管理、批量处理、排版预览和历史记录。

设计原则：

- 简洁

- 高效

- 一致

- 易学

- 易维护软件尽量减少学习成本，使首次使用即可完成排版任务。

---

# 二、设计原则细化

## 2.1 简洁

- 界面层级不超过三级。
- 避免弹窗，使用内联提示和状态栏。
- 同一功能不提供多个入口。

## 2.2 一致性

- 所有控件尺寸、间距、字体、颜色遵循统一设计令牌。
- 动画效果全局统一。
- 图标统一使用 Segoe Fluent Icons。

## 2.3 高效率

- 文件接支持拖拽，拖入即添加。
- 排版一键启动，无确认弹窗（除非存在覆盖风险）。
- 模板下拉即时生效，预览同步刷新。

## 2.4 非破坏性

- 默认输出原目录，文件名加 -R 后缀。
- 冲突时自动编号，绝不覆盖。
- 可自定义输出目录。

## 2.5 批量优先

- 参数全局生效，单文件错不影响其余任务。
- 递归添加文件夹（可选包含子文件夹）。

# 三、窗口布局

固定三栏布局，可适应窗口缩放（最小 960 × 840）。导航栏可自动折叠为图标模式。三个区域宽度比例约 1:2:2，支持拖拽分隔条，但结构长期稳定。

# 四、标题栏

- 高度固定 32px。
- 左侧：应用 Logo（32×32px），不显示软件名称。
- 中部：页面标签（如 “Word Formatter”），居中对齐。
- 右侧：Windows 原生最小化、最大化、关闭按钮。
- 标题栏背景跟随系统主题（浅色/深色），支持半透明效果。

---

# 五、导航栏

导航栏固定显示八个主要功能模块（带图标）：

```
页面设置

正文样式

标题样式

页眉页脚

图片设置

表格设置

高级设置

关于软件
```

点击后：

仅切换中间配置区域。

窗口其它区域保持不变。

交互细节：

- 当前选中项高亮（背景色 + 左侧指示条），点击其他项时平滑过渡。
- 键盘快捷键：Ctrl+1~8 可快速切换导航项。
- 当窗口缩窄时，导航栏自动收起为仅图标，鼠标悬停展开文字标签。

---

# 六、中间配置区域

始终显示当前导航模块对应的设置卡片，采用垂直排列、左对齐的 Card 布局。所有输入项格式统一：
[标签]  [输入框]  [单位选择器（可选）]

- 输入框高度 32px，标签使用系统默认字体（微软雅黑 9pt）。
- 单位选择器支持 cm、inch、pt、字符数等，与输入值联动。
- 配置项发生修改时，模块标题旁出现 “●” 表示未保存改动。
- 提供“保存为模板”和“重置为默认值”按钮，位于卡片底部。
- 当用户切换导航模块时，若当前模块有未保存修改，弹出轻量提示“是否保存对【模块名】的修改？”（可选“保- 存”“放弃”“取消”）。

---

# 七、右侧工作区域

右侧区域分为三个功能卡片，可折叠/展开。

## 7.1 文件管理卡片

提供文件列表及相关操作：

- 添加文件：点击打开文件选择器（可多选），支持 .doc、.docx。
- 添加文件夹：选择文件夹，可勾选“包含子文件夹”。
- 拖拽导入：从资源管理器直接拖文件或文件夹到本区域，自动添加。
- 列表操作：支持全选、反选、删除选中、清空列表；列表项显示文件名、路径（缩略）、文件大小；支持按住 Ctrl/Shift 多选。
- 文件总数实时更新，并在列表顶部显示“已选 N 个文件”。
- 最近打开：提供下拉菜单，列出最近使用的 10 个文件/文件夹，点击快速添加。
- 固定目录：用户可将常用目录固定到列表顶部，一键打开。
- 文件搜索：列表上方提供搜索框，实时过滤文件名。

## 7.2 排版控制卡片

- 输出目录：默认“原文件所在目录”，可更改为“指定目录”。点击浏览按钮选择。
- 模板选择：下拉列表选择已保存的模板，切换后配置区域立即更新。当前模板显示名称，若配置有未保存改动则名称旁显示 “*”。
- 开始排版：主操作按钮（强调色），点击后任务开始，按钮变为“取消任务”。
- 取消任务：中断当前任务，已处理的文件保留输出，未处理的文件不执行。
- 预览：在开始前可点击“预览效果”，展示当前配置的摘要清单或生成第一个文件的真实预览（详见第十三节）。
- 任务进度：显示进度条、百分比、当前处理文件名、成功/失败计数。
- 重试失败项：任务完成后，若有失败文件，提供“重试失败文件”按钮，仅对列表中的失败项重新排版。

## 7.3 结果与历史卡片

- 排版完成后自动显示结果摘要：成功 N 个，失败 M 个，耗时 X 秒。
- 提供“打开输出文件夹”和“查看失败文件”按钮。
- 失败文件列表支持逐个查看错误原因，并提供“再次尝试”操作。
- 历史记录入口：点击可展开最近 20 条任务历史，每条记录包含时间、模板、文件数、成功/失败数，支持一键复用该次任务的配置和文件列表。

---

# 八、状态栏

状态栏位于窗口底部，高度 32px，显示当前运行状态、文件计数、模板名称、软件版本。

状态类型及显示文本：

- 就绪：显示 “Ready | 已加载 N 个文件 | 模板：默认 | v1.2.0”
- 扫描中：显示 “正在扫描文件...”
- 排版中：显示 “处理中... (15/200) | 成功 12 | 失败 3”
- 完成：显示 “任务完成 | 成功 180 | 失败 20 | 耗时 1m23s”
- 失败：显示 “任务失败（用户取消或错误）”
- 错误：若内部错误导致服务中断，显示 “服务异常，请重启”
- 所有反馈优先显示在状态栏，避免弹窗打断用户。当需要用户确认时（如覆盖文件），使用非模态对话框或内嵌提示。

# 九、排版功能

每个功能模块提供独立的配置参数，具体如下：

## 9.1 页面设置

- 纸张大小：A4、Letter、B5 等标准尺寸 + 自定义宽高
- 页面方向：纵向 / 横向
- 页边距：上、下、左、右，独立输入，默认单位 cm
- 页码样式：无、顶部居中、底部居中、自定义位置（暂不支持复杂格式）

## 9.2 页眉页脚

- 中文字体、西文字体、字号、字形（加粗/倾斜/下划线）
- 对齐方式（左/中/右）
- 页眉距顶部距离、页脚距底部距离（cm）

## 9.3 正文样式

- 中文字体、西文字体、字号
- 字形（加粗/倾斜/下划线/删除线）
- 对齐方式
- 行距：单倍、1.5 倍、2 倍、固定值（pt）、最小值（pt）
- 段间距：段前间距、段后间距（pt 或 行）
- 缩进：首行缩进（字符/cm）、悬挂缩进、左右缩进

## 9.4 标题样式

覆盖 1～6 级标题，每一级可独立设置：

- 基于正文或独立字体、字号、字形
- 对齐方式
- 段前段后间距、行距
- 是否加编号（待实现）

## 9.5 图片设置

- 统一宽度（cm 或 “%” 页面宽度）
- 对齐方式（居中 / 左对齐 / 右对齐）
- 保持纵横比（勾选框）

## 9.6 表格设置

- 整体居中（勾选框）
- 自动分页（允许表格跨页断行）
- 表头字体（中/西）、字号
- 表格边框样式：无边框、全部框线、仅横向框线（基本选项）

# 十一、模板系统

模板用于保存和应用整套排版参数，支持完整的 CRUD 操作。

- 保存：当前配置可保存为新模板或覆盖现有模板（若为默认模板则另存为）。
- 导入：支持导入 .json 模板文件，文件需包含版本号以便兼容验证。
- 导出：将模板导出为 .json 文件，便于分享。
- 复制：基于当前模板创建副本。
- 删除：非默认模板可删除，默认模板不可删除但可重置。
- 重命名：支持模板重命名。
- 默认模板：软件自带“通用公文模板”“日常写作模板”等预置模板，用户可修改但永远保有恢复出厂设置的能力。
- 模板内容的版本控制遵循 tech-stack.md 中的配置 schema，保证向后兼容。

---

# 十二、文件输出

- 默认输出位置：原文件所在目录。
- 输出文件命名：原文件名-R.docx。若冲突则按 原文件名-R(1).docx、...-R(2).docx 递增。
- 用户可自定义输出目录，若选择“统一输出到指定文件夹”，则所有文件输出至该目录，命名规则同上。
- 若输出目录下存在同名文件，同样采取编号重命名策略，绝不覆盖。
- 支持 .doc 文件自动转换为 .docx 输出（通过后端 COM 转换），保留原 .doc 文件不变。

---

# 十三、预览

预览功能分为两个层级，以适应不同阶段的开发复杂度。

### Level 1：参数摘要预览（MVP 推荐）

- 在点击“预览”后，以卡片或列表形式展示当前所有设置项的汇总信息（如“正文：宋体 小四，行距 1.5 倍，首行缩进 2 字符”）。
- 用户快速浏览确认所有配置，无渲染成本。

### Level 2：真实排版预览（后续版本）

- 选取文件列表中第一个文件作为样本，在后台生成临时排版后的 docx，通过 Office 文档查看控件或导出为图片展示在预览窗口。
- 支持切换预览页面、标题等元素。
- 预览窗口在点击“开始排版”后自动关闭，亦可手动关闭。

当前阶段（V2.0）建议先实现 Level 1，确保用户能快速理解将要应用的效果。

---

# 十四、历史记录

历史记录包含近期任务和近期使用：

- 最近任务：保存最近 20 次排版任务，每条记录包含：任务开始时间、模板名称、文件数量、成功/失败计数、输出目录。用户可从历史记录中一键“重用配置”（加载模板及文件列表）或“打开输出目录”。
- 最近打开文件/文件夹：记录最近 10 个添加的文件或文件夹路径，便于快速再次添加。
- 最近使用模板：记录最近 5 个使用的模板，方便切换。
- 历史数据存储在本地 SQLite 中（详见技术栈文档）。

---

# 十五、性能要求

支持：

- 支持 1000 个以上 Word 文档连续排版，UI 线程不被阻塞。
- 排版过程中界面保持响应：用户可随时取消任务、切换页面查看进度。
- 进度更新间隔不超过 200ms（每处理完 1 个文件或每 200ms 刷新一次）。
- 大文件（>50MB）单个处理耗时较长时，应显示正在处理的文件名，避免“假死”。
- 启动时间：冷启动 < 2 秒，热启动（服务已驻留）< 0.5 秒。
- 内存占用：处理 500 个文件时，内存增长不超过 200MB，且任务完成后内存回收。

# 十六、异常处理

- 单文件异常：单个文件处理失败（如文件损坏、格式不兼容、COM 异常）时，记录失败原因到错误列表，继续处理剩余文件。
- 任务取消：用户取消后，已完成的文件保留输出，未处理的文件列表可保留供后续重试。
- 全局错误：如后端服务崩溃、磁盘空间不足等，及时在状态栏显示错误信息，并提示用户可采取的措施（如清理磁盘、重启服务）。不弹出阻塞性对话框。
- 错误报告：失败文件列表提供详细的错误描述（如“第 3 页表格格式异常”），支持导出错误日志（.txt）以便用户反馈。
- 重试机制：任务完成后，界面提供“重试失败项”按钮，仅对失败的子集重新排版，无需重新添加文件。

---

# 十六之二、日志与归档

- 日志按用途分文件：`app.log`（前端/UI）、`backend.log`（API/Service）、`format.log`（排版任务）、`error.log`（所有异常汇总）。用户反馈问题时可直达对应文件，无需在海量日志中翻找。
- 日志文件自动归档：每日午夜滚动，历史日志按日期归档，用户无需手动清理。
- 保留策略：`logs/` 根目录的每日文件保留最近 30 天，之后移入 `logs/archive/`；归档日志默认保留 90 天，到期自动删除。
- 日志由后端统一产生，前端不直接写日志；全程无弹窗打扰用户。

# 十七、视觉规范

整体风格遵循 Windows 11 Fluent Design System。

- 圆角：控件和容器默认圆角半径 8px；小型按钮 4px。

- 阴影：卡片悬浮时显示柔和阴影（Elevation 4），无硬边框。

- 半透明效果：标题栏、导航栏背景使用 Mica 或 Acrylic 材质，随主题变化。

- 颜色系统：
  
   - 主色调：蓝色系（#0078D4），用于强调按钮、选中状态。
   - 语义色：成功绿（#13A10E）、警告橙（#FF8C00）、错误红（#D13438）。
   - 文本色：主要文本 #1A1A1A（浅色主题）或 #FFFFFF（深色主题）；次要文本 #666666。
   - 背景：浅色主题 #F3F3F3，深色主题 #202020；卡片背景浅色 #FFFFFF，深色 #2D2D2D。
   - 字体：全局统一使用 Segoe UI Variable，代码或数字使用 Cascadia Mono。正文字号 9pt，标题字号 11pt。
   - 间距：标准内边距 12px，卡片之间间距 16px，区域分割线 1px 浅灰色。
   - 深色/浅色主题：跟随系统设置或手动切换，所有颜色自动适配。

# 十八 国际化

多语言预留：初期仅支持简体中文，但代码中所有用户可见字符串必须集中管理（如资源文件），为后续国际化做准备。UI 控件尺寸需考虑英文等语言文本长度变化（如最长字符串 +30% 空间）。
